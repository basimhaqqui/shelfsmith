# ShelfSmith

A serverless AI **product-content engine** for an e-commerce electronics catalog
(audio gear, appliances, vehicle accessories). POST a raw product (title + a few
specs) and it returns a polished marketing description, a category, bullet-point
features, and SEO keywords — generated by **Amazon Bedrock (Claude Haiku 4.5)**
and persisted to **DynamoDB**. A scheduled job summarizes customer-review
sentiment into a digest.

## Live demo

- **Web UI:** https://dexsky4i7f48e.cloudfront.net
- **API (enrich):** `POST https://xagag1olfg.execute-api.us-east-1.amazonaws.com/enrich`

**90-second demo flow:**
1. Open the web UI → it's on the **Generate** tab with a product pre-filled.
2. Click **Generate content** → watch Bedrock return a description, category, feature bullets, and SEO keywords live.
3. Switch to the **Catalog** tab → the product you just made is listed (served by `GET /products`).
4. In a terminal, run the scheduled job on demand: `./demo.sh` (or invoke the digest Lambda) to show the review-sentiment summary.

## Architecture

```
                         POST /enrich
   client  ────────────────────────────────►  API Gateway (HTTP API)
                                                      │
                                                      ▼
                                               Lambda: enrich
                                                      │
                                          ┌───────────┴───────────┐
                                          ▼                       ▼
                                   Amazon Bedrock           DynamoDB table
                                 (Claude Haiku 4.5)      (single table, keyed
                                                          by productId + sk:
                                                          PRODUCT / REVIEW# /
                                                          DIGEST)
                                                              ▲
   EventBridge (rate: 1 day) ──► Lambda: review-digest ──────►│
                                  (scan reviews → Bedrock      │
                                   sentiment summary → put)────┘
```

## AWS services & why

| Service | Role | Why |
|---|---|---|
| **API Gateway (HTTP API)** | Public `POST /enrich` | Cheaper + simpler than REST API; native Lambda proxy |
| **AWS Lambda** | `enrich`, `review-digest` | Pay-per-invoke, zero idle cost |
| **Amazon Bedrock** | Claude Haiku 4.5 (Converse API) | Managed LLM, no servers; fast + cheap model |
| **DynamoDB** | Persist products, reviews, digests | Serverless, on-demand billing, single-table design |
| **EventBridge** | Daily trigger for the digest | Native cron, no server |
| **AWS CDK (TypeScript)** | Infrastructure as code | One language for infra + handlers; esbuild-bundled Lambdas |

**Least-privilege IAM:** each Lambda gets only what it needs — `enrich` =
`bedrock:InvokeModel` (scoped to the model + inference-profile ARNs) +
`dynamodb:PutItem`; `review-digest` = `bedrock:InvokeModel` + `dynamodb:Scan` +
`dynamodb:PutItem`. No wildcards.

## Deploy

```bash
npm install
npx cdk bootstrap                 # once per account/region
npx cdk deploy                    # prints the API URL + resource names
```

## Seed demo data

```bash
npm run seed                      # 5 sample products + ~17 mock reviews
```

## Test

```bash
# 1. Enrich a product (returns structured JSON + persists it)
ENRICH_URL="$(aws cloudformation describe-stacks --stack-name ShelfSmithStack \
  --query "Stacks[0].Outputs[?OutputKey=='EnrichEndpoint'].OutputValue" --output text)"

curl -s -X POST "$ENRICH_URL" -H 'content-type: application/json' \
  -d '{"title":"AeroBass X2 Wireless Earbuds","specs":"BT 5.3, ANC, IPX5, USB-C"}' | python3 -m json.tool

# 2. Run the review digest now (instead of waiting for the daily schedule)
DIGEST_FN="$(aws cloudformation describe-stacks --stack-name ShelfSmithStack \
  --query "Stacks[0].Outputs[?OutputKey=='DigestFunctionName'].OutputValue" --output text)"

aws lambda invoke --function-name "$DIGEST_FN" \
  --cli-binary-format raw-in-base64-out --payload '{}' /tmp/digest.json
cat /tmp/digest.json | python3 -m json.tool
```

## Tear down

```bash
npx cdk destroy                   # removes all stack resources incl. the table
```

> Then delete the demo IAM user/access key you created for the CLI, and
> remember Bedrock inference is the only non-free cost.

## Cost

Comfortably inside the AWS free tier for a demo. DynamoDB is on-demand (≈$0
idle), Lambda is pay-per-invoke, API Gateway HTTP API is fractions of a cent
per call. The only real charge is Bedrock inference — Claude Haiku 4.5 is a few
**cents** across a demo's worth of calls. **Run `cdk destroy` when done.**
